# sd2-2: Bash availability probe ignores effective warm command resolution

**Severity**: LOW — wrong-behavior recovery advice only in warm sessions
that shadow `bash`; exotic but deterministic when triggered.
**Status**: Verified
**Branch**: master (direct, per recorded precedent)
**Commit**: `168d027`

## Evidence
`server/PtkMcpServer/RunspaceHost.cs` (ProbeBashAvailable) resolves `bash`
with an Application-filtered lookup. A warm `function bash { ... }` (or
alias) shadows the application under normal command precedence, so the
advice "wrap it in bash -lc" would invoke the shadow, not bash.
Master-verified 2026-07-09: with the function defined, the Application
lookup still returns Application while effective resolution is Function,
and `bash -lc 'echo hi'` runs the function ("SHADOWED:").

## Predicted observable failure
On a bash-equipped box with a warm shadow named `bash`, the refusal offers
the `bash -lc` wrap; re-issuing it executes the shadow with the script as
an argument — silently wrong behavior on the advertised recovery path.

## What
The probe answers "does a bash Application exist anywhere" when the
question the advice depends on is "what would `bash` actually run here" —
the same resolution-context discipline the plan enforces everywhere else.

## Approach
Resolve `bash` with normal precedence (CommandTypes::All) and offer the
wrap only when the *effective* command is an Application. Cold context is
unaffected (nothing shadows in the pristine detection runspace).

## Files changed
- `server/PtkMcpServer/RunspaceHost.cs` — ProbeBashAvailable resolves with
  CommandTypes::All and requires the effective command's CommandType to be
  Application.
- `server/PtkMcpServer.Tests/ShellDialectWiringTests.cs` — new
  Warm_bash_shadow_withdraws_the_wrap_advice: warm `function bash`, then a
  detected script → refusal present, rewrite advice present, no `bash -lc`
  offer. Deterministic on bash-present and bash-absent boxes alike.

## Guard proof
- `ShellDialectWiringTests::Warm_bash_shadow_withdraws_the_wrap_advice` —
  red leg run 2026-07-09 on this bash-equipped Mac: with the probe fix
  stashed, FAILS (wrap still offered under the shadow); restored, PASSES
  (class suite 12/12). On a bashless box the red leg is vacuous by
  construction (advice is already rewrite-only) — noted, not concealing.

## Coder dispute (if any)
None — admitted; shadow behavior verified by the coder before triage.

## Known gaps
An alias pointing AT the real bash is treated as unavailable (CommandType
Alias ≠ Application) — conservative under the plan's "offered only when
bash resolves as an Application" wording; the rewrite path always remains.
